# Example: Solo Sharoushi Office

1人社労士事務所向けの例です。

## Common Problems

- 問い合わせ対応に時間が取られる
- 顧問先ごとの事情が頭の中にある
- 手続き、相談、提案の優先順位が混ざる
- WebサイトやLINEからの導線が業務フローとつながっていない
- AIを使っても、最終確認の基準が曖昧

## AI Employee Candidates

| Role | AI tasks | Human tasks |
|---|---|---|
| 問い合わせ整理AI | 要約、分類、不足情報の洗い出し | 最終回答、例外判断 |
| 顧問先メモAI | 面談メモ整理、次回対応案 | 提案判断、関係構築 |
| FAQ下書きAI | よくある質問の整理、回答案 | 法的確認、公開判断 |
| Web導線改善AI | ページ構成案、CTA案、FAQ案 | 事務所方針、表現判断 |

## First Workflow To Build

最初に作るべきなのは、問い合わせ対応のAI補助フローです。

理由は、頻度が高く、文章処理が多く、AIの効果が見えやすく、人間の最終確認も入れやすいからです。
