# Market-In Principles

このプロジェクトは、技術起点ではなく、市場と顧客課題から逆算して設計します。

## Target Users

- 人手不足に悩む中小企業
- 業務が属人化している小規模事業者
- 問い合わせ対応や文書作成が重い1人社労士
- WordPress、LINE、Web導線を支援する実務者
- AI活用を現場に落とし込みたい伴走支援者

## Market Problems

- 人を採用できない
- 教育に時間がかかる
- 業務が人に依存している
- AIを試しても継続運用できない
- 現場の業務と言語化された手順がつながっていない

## Reverse-Engineered Requirements

市場課題から逆算すると、OSSに必要な要素は次の通りです。

| Requirement | Reason |
|---|---|
| 業務分解 | AI化できる作業を見つけるため |
| 役割定義 | AI社員を曖昧な存在にしないため |
| 責任境界 | 人の判断とAIの処理を分けるため |
| 運用手順 | 単発利用で終わらせないため |
| 改善ログ | 継続的に精度と実用性を上げるため |
